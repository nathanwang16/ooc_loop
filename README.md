# ooc_loop
The engineering research project for automation processes of organ on chip system optimization and production

## Revision History

- **v0.1.0** — Scaffolded full project file structure reflecting all 5 pipeline phases and module dependencies (geometry, cfd, config, optimization, analysis, validation). Added OpenFOAM template case, default YAML config, entry-point scripts, and test stubs.
- **v0.2.0** — Module 1.1 complete: OpenFOAM solver verification. Implemented Poiseuille flow analytical solution, blockMeshDict generator, OpenFOAM field parser, automated verification script, and mesh convergence study. Verified <2% error at all refinement levels. Added collaboration tooling (environment.yml, pyproject.toml, Makefile, conftest.py).
- **v0.3.0** — Module 3.1 implemented: completed single-configuration BO runner and 8-configuration orchestrator with constrained objective handling, Sobol initialization, GP fitting, and run-state serialization. Added concrete constraint formulation tests in `tests/test_optimization.py`.
- **v0.4.0** — Implemented pillar-case geometry/meshing and analysis utilities: refactored CadQuery generator with validation and pillar-obstacle STL export, upgraded CFD setup for robust inlet/patch injection and pillar meshing workflow, and completed M3.2 plotting/report scripts (`convergence`, `comparison`, `wss_contours`, `run_analysis.py`) with smoke-tested figure generation.
- **v0.5.0** — Hardened pillar meshing quality handling: removed degraded blockMesh fallback, added per-case meshing diagnostics/log files and checkMesh summaries, exposed snappy/mesh quality thresholds in config, and enforced BO penalization plus infeasible filtering for low-quality meshes (`mesh_ok=False`).
- **v0.6.0 (v2 pivot, Modules 1.1–3.3)** — Pivoted the project from WSS uniformity to inverse design of tumor-on-chip drug-gradient geometry with interpretability output, per Development_Guide_v2.md:
  - **M1.1**: added `scalarTransportFoam` template (`0/T`, `DT` in `transportProperties`, `div(phi,T)` scheme, `T` solver block) and a 1D advection-diffusion verification module (`ooc_optimizer/cfd/scalar.py`, `scripts/run_scalar_verification.py`).
  - **M1.2**: revised CadQuery generator for two-inlet chips across three inlet topologies (`opposing`, `same_side_Y`, `asymmetric_lumen`) and added a topology-aware multi-block `blockMeshDict` generator (`ooc_optimizer/geometry/topology_blockmesh.py`) enforcing the v2 patch-naming contract (`inlet_drug`, `inlet_medium`, `outlet`, `walls`, `floor`, `frontAndBack`).
  - **M2.1–2.2**: refactored the CFD orchestrator into a two-step solve (`simpleFoam` → `scalarTransportFoam`) with new metrics (`L2_to_target`, `grad_sharpness`, `monotonicity`) and target-profile callables (`linear_gradient`, `bimodal`, `step`, `custom`) in `ooc_optimizer/optimization/objectives.py`.
  - **M2.3**: replaced dict-based schema with pydantic v2 models (`ooc_optimizer/config/schema.py`), extended the JSONL evaluation logger to carry `inlet_topology`, `target_profile`, and both v2 and v1 objective keys.
  - **M3.1**: extended `BORunner` to minimise `L2_to_target` subject to relaxed WSS + dead-zone constraints, with a 7D parameter vector and per-topology active-parameter masks; added `run_multi_target_workflow` that sweeps 24 discrete configurations for the primary target and only the winning topology for the two secondary target profiles.
  - **M3.2**: replaced WSS contour plots with concentration-field plots (contour, residual, centerline, streamline overlay, BO convergence, winner grid) in `ooc_optimizer/analysis/concentration_fields.py`.
  - **M3.3 (NEW)**: added `ooc_optimizer/interpretability/{sobol.py, gp_gradients.py, tolerance.py, pipeline.py}` implementing SALib-based Sobol indices on the trained GP, autograd-based local GP-gradient sensitivities at the optimum, and bisection-based tolerance intervals per parameter. Ships a single-command driver `scripts/run_interpretability.py` that produces per-run figures (`sobol.png`, `local_sensitivity.png`, `tolerance.png`) and auto-writes a `design_heuristics.md` summary per winner.
  - **M4.1 (3D CFD validation)**: added `ooc_optimizer/geometry/topology_blockmesh.py::generate_blockmesh_dict_v2_3d` (z-extruded multi-block 3D meshes with `floor`/`ceiling` patches), `ooc_optimizer/validation/cfd_3d_v2.py::validate_winner_3d` (3D `simpleFoam` + `scalarTransportFoam` on a BO winner reporting `L2_to_target_3d`, `L2_relative_delta`, and floor-WSS statistics), `ooc_optimizer/validation/compare_plots_v2.py` (three figures: concentration residual, centerline overlay, WSS scatter + Bland-Altman), and a rewritten `scripts/run_3d_validation.py` that accepts both a single BO state directory and a multi-target orchestrator summary.
  - **M5.2 (packaging and repository standards)**: rewrote `pyproject.toml` around the hatchling backend with pinned dependency ranges, `dev` / `test` / `docs` optional extras, full ruff + mypy + pytest-cov configuration, and a `tumor-chip` console script. Added a `tumor_chip_design` compatibility namespace package so `import tumor_chip_design` works after `pip install tumor-chip-design` (the underlying implementation remains `ooc_optimizer`; full src-layout migration is scheduled for v0.7.0). Ships `LICENSE` (MIT), `CITATION.cff`, `CHANGELOG.md`, `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `.pre-commit-config.yaml`; GitHub workflows for CI (ruff + mypy + pytest matrix on 3.10/3.11/3.12), docs (MkDocs Material → Pages) and release (PyPI trusted publishing + Zenodo integration); issue and PR templates; a `docker/Dockerfile` layering Python 3.11 on `opencfd/openfoam-default:2406` plus `docker-compose.yml`; an `mkdocs-material` docs skeleton with auto-generated API reference via `mkdocstrings`; a JOSS companion paper draft (`paper/joss/paper.md` + `paper.bib`); and `examples/{tumor_chip_linear_gradient, tumor_chip_bimodal, wss_uniformity}` example directories — the last retains the full v1 worked example per Development Guide v2 §0.
