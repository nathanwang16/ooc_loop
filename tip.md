# Tips & Bug Notes

- **OpenFOAM macOS**: `codedFixedValue` fails on DMG-mounted OpenFOAM (macOS .app bundle) because the dynamic code compiler can't write to the mounted volume. Use `fixedValue` with uniform inlet instead; the flow develops within L_dev ≈ 0.2 mm on a 10 mm channel.
- **OpenFOAM macOS install**: Use `brew tap gerlero/openfoam && brew install gerlero/openfoam/openfoam@2406`. Commands are accessed via the `openfoam2406` wrapper (e.g. `openfoam2406 -c 'simpleFoam -case ...'`).
- **turbulenceProperties**: `simpleFoam` requires `constant/turbulenceProperties` even for laminar flow — set `simulationType laminar;`.
- **numpy 2.x**: `np.trapz` removed; use `np.trapezoid`.
- **CAD dependency coupling**: Importing `ooc_optimizer.cfd` transitively imports geometry (`cadquery`). BO/CFD tests that import `ooc_optimizer.cfd.*` require `cadquery` installed even if geometry generation is not exercised.
- **macOS inplace edits**: Avoid `sed -i` for OpenFOAM case file updates on macOS. Use Python string replacement to patch `0/U` values safely.
- **Pillar meshing quality policy**: `snappyHexMesh` can still produce low-quality thin-slab meshes (e.g., concave cells). Pipeline no longer falls back to background `blockMesh`; it keeps the snappy mesh, logs `checkMesh` diagnostics, marks `mesh_ok=False`, and BO applies a penalty objective.
- **Manual STL+CFD check**: `source /Volumes/OpenFOAM-v2406/etc/bashrc` then `conda activate ooc` and run `python scripts/run_single_verification.py` from repo root (`--complex` for 3x6 pillars, H=300 um). Outputs under `data/manual_verification/` or `data/manual_verification_complex/`; see each folder's `VERIFICATION_MANIFEST.md`.
- **ParaView OpenFOAM loading**: Opening an OpenFOAM case directory via `All Files` can trigger the wrong reader (for example `vtkADIOS2CoreImageReader`) and show an empty view. Create a marker file with `touch <case>/<name>.foam`, then open that `.foam` file in ParaView and choose `OpenFOAMReader`/`POpenFOAMReader`.
- **Convergence threshold mismatch**: `ooc_optimizer/cfd/template/system/fvSolution` sets `residualControl` to `1e-6`, while `ooc_optimizer/cfd/solver.py` `_check_convergence` defaults to `1e-4`. Premise-test runs force `1e-6` for both solver controls and post-checking to avoid CV-comparison noise from loose residual gates.
- **v2 scalarTransportFoam pattern**: after `simpleFoam` converges in the same case directory, `ooc_optimizer.cfd.scalar.run_scalar_transport` swaps `system/controlDict` to `application scalarTransportFoam` and relies on `ddtSchemes.default steadyState` to drive a steady scalar solve on the frozen U/phi fields. Do not reset the case between the two solves; scalarTransportFoam needs `phi` from the latest time directory.
- **v2 patch-naming contract**: geometry / blockMesh / solver all expect the patches `inlet_drug`, `inlet_medium`, `outlet`, `walls`, `floor`, `frontAndBack`. Topology-aware blockMeshDict lives in `ooc_optimizer.geometry.topology_blockmesh` and is the single source of truth for these names.
- **v2 BO dims**: `PARAMETER_ORDER` = [W, d_p, s_p, theta, Q_total, r_flow, delta_W] for *all* topologies; inactive slots are pinned to 0.5 and the active mask is stored with the GP state so interpretability can skip them.
- **3D validation mesh**: `generate_blockmesh_dict_v2_3d` extrudes the 2D multi-block layout in z with `nz` layers and replaces `frontAndBack=empty` with explicit `floor` (z=0) / `ceiling` (z=H) patches.  Use `z_grading < 1` to concentrate cells near the floor for resolved `wallShearStress`; leave at 1.0 for quick sanity runs.
- **tumor-chip CLI**: `tumor-chip {verify-scalar,optimize,interpret,validate-3d,version}` wraps the scripts in `scripts/`.  The CLI is installed by `pip install -e ".[test]"` via the console-script entry in `pyproject.toml` and requires `typer` (already pinned in the package dependencies).
- **scalarTransportFoam high-Pe verification**: Three failure modes surfaced for the Pe=1000 case in `run_scalar_verification_1d`: (a) template `div(phi,T) bounded Gauss limitedLinear 1` → SIGFPE at high cell Pe; switched to `bounded Gauss upwind`; (b) template `fvSolution.T` uses `symGaussSeidel` smoother which cannot converge advection-dominated matrices; switched to `PBiCGStab` + `DILU`; (c) first-order upwind is excessively diffusive for cell_Pe > 5 → auto-refine `n_cells` so `cell_Pe ≤ 2` (i.e. `n_cells_eff = max(user_n, ceil(Pe/2))`). The 1D verification case uses a 1 mm cross-section (not 10 μm) to keep `phi` well above float64 noise.
- **Package names**: after `pip install tumor-chip-design` both `import ooc_optimizer` and `import tumor_chip_design` work; the latter is a thin re-export shim in `tumor_chip_design/__init__.py`.  Full src-layout migration is tracked in CHANGELOG.md as the v0.7.0 milestone.
